## How to Use These Tests

### Run All Tests
```bash
npm test
```

### Run Specific Test File
```bash
npm test -- --include='**/auth.service.spec.ts'
npm test -- login.spec.ts
```

### Run Tests in Watch Mode
```bash
npm test -- --watch
```

### Generate Coverage Report
```bash
npm test -- --code-coverage
```

### Run Tests Without Watch
```bash
npm test -- --no-watch
```

### Run in Headless Mode
```bash
npm test -- --browsers=ChromeHeadless
```

## Test Naming Convention

All tests follow Angular/Jasmine conventions:

```typescript
describe('Component/Service Name', () => {
  describe('Feature/Method', () => {
    it('should [expected behavior]', () => {
      // Test implementation
    });
  });
});
```

### Examples
- `should login successfully with valid credentials`
- `should handle 401 Unauthorized error`
- `should toggle password visibility`
- `should require email validation`
- `should update signal on change`

## Assertion Patterns Used

### Type Checking
```typescript
expect(component).toBeTruthy();
expect(value).toBe(expected);
expect(value).toEqual(expected);
```

### Array Testing
```typescript
expect(array.length).toBe(3);
expect(array).toContain(item);
```

### HTTP Testing
```typescript
const req = httpMock.expectOne(url);
expect(req.request.method).toBe('POST');
req.flush(response);
```

### Observable Testing
```typescript
service.method().subscribe({
  next: (result) => expect(result).toBeTruthy(),
  error: done.fail
});
```

### Signal Testing
```typescript
service.signal.set(value);
expect(service.signal()).toBe(value);
```

### Form Testing
```typescript
expect(form.valid).toBe(true);
expect(control.hasError('required')).toBe(true);
```

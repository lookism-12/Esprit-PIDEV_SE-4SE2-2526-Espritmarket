                      HOW TO RUN THE TESTS

FRONTEND (Angular):

  # Run all frontend tests
  $ cd frontend
  $ npm test

  # Run specific test file
  $ npm test -- login.comprehensive.spec.ts
  $ npm test -- register.comprehensive.spec.ts
  $ npm test -- auth.service.comprehensive.spec.ts

  # Run with coverage report
  $ npm test -- --coverage

  $ npm test -- --watch

  $ cd backend
  $ mvn test

  # Run specific test class
  $ mvn test -Dtest=UserServiceImprovedTest

  # Run with coverage

FRONTEND VALIDATIONS:

Login Form:
  - email: [required, email format]
  - password: [required]
  - rememberMe: [optional checkbox]

Register Form:
  - firstName: [required, minLength(2)]
  - lastName: [required, minLength(2)]
  - email: [required, email format]
  - password: [required, minLength(8)]
  - confirmPassword: [required, must match password]
  - phone: [required, pattern(/^[0-9]{8,15}$/)]
  
  Role-Specific:
  - Client: optional address
  - Provider: businessName, businessType, taxId (required)
  - Driver: vehicleType, drivingLicenseNumber (required)
  - Delivery: vehicleType, deliveryZone (required)

BACKEND VALIDATIONS:

Avatar Upload:
  - File not empty
  - MIME type must be image/*
  - File size ≤ 10MB
  - Supported formats: JPEG, PNG, GIF, WebP

Password Reset:
  - Token must be valid (exists in database)
  - Token must not be expired (1 hour expiry)
  - New password must be encoded before save

Profile Update:
  - firstName: only updated if not null/blank
  - lastName: only updated if not null/blank
  - phone: only updated if not null/blank
